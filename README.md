# Growth Opportunity Mining

> 从公开评价、社区讨论、评测与竞品证据中，识别产品增长机会与用户需求。

一个用于 Kimi Work / Kimi 智能体的结构化工作流 Skill，帮助你在研究新品类、分析竞品、挖掘用户痛点时，保持方法一致、证据可查、结论可落地。

---

## 适用场景

- 研究一款产品或品类是否值得进入
- 对比多款竞品，找产品差异化机会
- 从小红书、电商、社区、评测中提炼用户真实反馈
- 输出可交付的增长分析报告和可视化网页

---

## 核心能力

| 能力 | 说明 |
|---|---|
| **结构化采集** | 按产品/竞品拆分，从电商、社区、评测、投诉等渠道收集证据 |
| **主题归因** | 将零散评价归一到购买动机、使用场景、满意点、痛点、未满足需求等维度 |
| **需求优先级矩阵** | 按频率、情绪强度、影响人群判断哪些需求最值得投入 |
| **竞品横向对比** | 定位、人群、动机、场景、优势、槽点、机会一表对比 |
| **报告 + 可视化网页** | 最终输出 Markdown 报告 + 可交互网页看板，含真实证据卡片 |

---

## 标准流程（10 步）

1. **确认研究范围** — 明确品类、竞品、时间、语言、目标输出
2. **建立目录结构** — 规范存放原始数据、分析结果、证据、报告、网页
3. **采集证据** — 不编造、不绕过登录，受限来源标注置信度
4. **合并清洗** — 评估样本质量、来源分布、置信度比例
5. **主题归因** — 归一化标签，识别高频需求与痛点
6. **横向对比** — 生成竞品对比表与机会矩阵
7. **报告预览** — 先确认结构与核心结论，再生成全文
8. **正式报告** — 结论前置，每条主结论附双来源证据
9. **可视化网页** — 图表 + 筛选 + 真实评论证据卡片
10. **验收修复** — 检查布局、图表、筛选器、溢出、移动端适配

## 质量红线

- **不编造评论** — 受限来源改用公开可访问替代，并标注置信度
- **不隐藏限制** — 反爬、登录、样本不足如实说明
- **双来源支撑** — 关键结论至少引用 2 条不同来源证据
- **结论与证据分离** — 原始数据、分析结果、最终报告分目录存放
- **先确认再交付** — 报告结构和核心结论预览后，再生成网页
